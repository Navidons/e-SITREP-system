### **Document 1: Executive Summary & Project Pitch (1-Page Version)**

**Project Title:** Border SITREP Automation System (BSAS)

**Overview**  
The BSAS is a secure, scalable web-based platform that automates the daily and weekly Situation Reports (SITREPs) for over 60 border posts under the National Citizenship and Immigration Control (NCIC), Ministry of Internal Affairs.

**Problem Statement**  
Currently, officers at each border post manually prepare daily reports which are sent to Headquarters. HQ staff then spend significant time manually compiling them into one consolidated SITREP and weekly Excel reports. This process is time-consuming, error-prone, and delays critical decision-making.

**Proposed Solution**  
A role-based web portal where:
- Station officers enter data easily.
- HQ teams review, verify, and approve.
- The system automatically generates professional Daily Consolidated SITREPs and Weekly Reports.

**Key Benefits**  
- Reduces report compilation time from hours to minutes  
- Eliminates manual errors  
- Full audit trail and accountability  
- Offline capability for remote borders  
- Secure, compliant with government standards  

**Alignment with Government Priorities**  
Interoperability & Data Exchange, Monitoring & Evaluation Systems, Justice Law & Order, Digital Transformation.

**Prototype Readiness**  
MVP ready by 1st June 2026 for the Showcase on 25th June 2026.

### **Document 2: System Requirements Specification (SRS)**

**1. Functional Requirements**

**FR-01: User Management**  
- Admin can create users and assign roles (ADMIN, HQ_AUTHORISER, HQ_VERIFIER, HQ_REVIEWER, STATION_INPUTTER, CLUSTER_SUPERVISOR)

**FR-02: Station Data Entry**  
- STATION_INPUTTER can enter arrivals, departures (by nationality code, male/female), special categories, incidents, remarks  
- Auto-calculation of totals  
- Offline mode with synchronization

**FR-03: Workflow & Approval**  
- Multi-level approval: Submitted → Reviewed → Verified → Approved

**FR-04: Report Generation**  
- Generate Individual Station Daily Report (PDF)  
- Generate Consolidated National Daily SITREP (exact current format)  
- Generate Weekly Excel Summary (Stations × Days + Totals)

**FR-05: Dashboard & Analytics**  
- Role-based dashboards  
- Historical search and trends

**2. Non-Functional Requirements**  
- Security: RBAC + Row Level Security + Audit Trail  
- Performance: Support 60+ concurrent stations  
- Usability: Intuitive interface, mobile responsive, PWA  
- Availability: 99% uptime, offline support  
- Scalability: Horizontal scaling  
- Compliance: Uganda Data Protection Act

### **Document 3: Software Architecture Document (High-Level Design)**

**Architecture Style:** Layered + API-First (Microservices-ready)

**Layers:**
1. **Presentation Layer** – React.js + TypeScript + PWA
2. **Business Logic Layer** – NestJS (TypeScript)
3. **Data Access Layer** – PostgreSQL with Prisma/TypeORM
4. **Reporting Layer** – Puppeteer (PDF) + exceljs
5. **Integration Layer** – REST APIs (future PISCES/MIDAS)
6. **Security Layer** – Keycloak / JWT + PostgreSQL RLS

**Data Flow (Daily):**
Station Input → Validation → Submit → HQ Review → Approval → Auto Generate Consolidated SITREP → Archive & Notify

**Deployment Architecture:**
- Docker containers
- On-premise Kubernetes cluster (primary)
- Hybrid backup

### **Document 4: Database Design Document**

**Key Tables (PostgreSQL)**

- `border_stations` (id, code, name, cluster, type)
- `users` (id, username, full_name, station_id, is_active)
- `roles`, `permissions`, `user_roles`, `role_permissions`
- `station_daily_reports` (id, station_id, report_date, status, submitted_by…)
- `movements` (report_id, movement_type, nationality_code, male, female)
- `special_categories`, `incidents`
- `audit_logs`

**Important Features:**
- Generated columns for totals
- Materialized views for weekly aggregation
- Row Level Security policies per role
- Audit triggers

### **Document 5: API Specification (OpenAPI Style Summary)**

**Key Endpoints:**

- `POST /auth/login`
- `POST /reports/station` – Create station daily report
- `GET /reports/pending` – HQ view pending reports
- `POST /reports/:id/review`
- `POST /reports/generate/consolidated?date=YYYY-MM-DD`
- `GET /exports/station/:id/pdf`
- `GET /exports/weekly?from=...&to=...`

All endpoints secured with JWT and role-based guards.
