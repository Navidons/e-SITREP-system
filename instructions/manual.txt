### **Updated System Architecture (Enterprise, Secure, Scalable)**

#### **1. Core Database Design (PostgreSQL) – Optimized for this exact flow**

```sql
-- 1. Master Stations
CREATE TABLE border_stations (
    id SERIAL PRIMARY KEY,
    code VARCHAR(20) UNIQUE,
    name VARCHAR(100) UNIQUE NOT NULL,
    cluster VARCHAR(50),
    type VARCHAR(30), -- Land, Air, Lake, etc.
    active BOOLEAN DEFAULT true
);

-- 2. Daily Station Report (one per station per day)
CREATE TABLE station_daily_reports (
    id SERIAL PRIMARY KEY,
    station_id INTEGER REFERENCES border_stations,
    report_date DATE NOT NULL,
    submission_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    submitted_by VARCHAR(100),
    status VARCHAR(20) DEFAULT 'submitted', -- submitted, reviewed, approved
    staff_on_duty INTEGER,
    medical_screening TEXT,
    general_remarks TEXT,
    urgent_matters TEXT,
    UNIQUE(station_id, report_date)
);

-- 3. Movements (Normalized - best for aggregation & accuracy)
CREATE TABLE movements (
    id SERIAL PRIMARY KEY,
    report_id INTEGER REFERENCES station_daily_reports,
    movement_type VARCHAR(10) NOT NULL CHECK (movement_type IN ('arrival','departure')),
    nationality_code VARCHAR(10),   -- BI, KE, SSD, UG, etc.
    male INTEGER DEFAULT 0,
    female INTEGER DEFAULT 0
);

-- 4. Special Categories
CREATE TABLE special_categories (
    report_id INTEGER REFERENCES station_daily_reports,
    category VARCHAR(50), -- asylum_seekers, deported, offloaded, etc.
    male INTEGER, female INTEGER, details TEXT
);

-- 5. Incidents / Remarks
CREATE TABLE incidents (
    id SERIAL PRIMARY KEY,
    report_id INTEGER,
    incident_type VARCHAR(100),
    description TEXT,
    passport_no VARCHAR(50),
    action_taken TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**Why this structure?**
- Easy per-station data entry.
- Extremely fast daily/weekly aggregations.
- Maintains 100% fidelity to current output format.
